```typescript
export interface ValidationResult {
  pass: boolean
  explanation: string
  columnDiff?: { missing: string[]; extra: string[] }  // optional diff details
}

export function validateAnswer(
  userAnswer: TableAnswer[],
  correctAnswer: TableAnswer[]
): ValidationResult { ... }
```

Validation logic (set-based, per D-01/D-02):
1. Check table count: `userAnswer.length === correctAnswer.length`. If not → fail with "Expected {N} table(s), got {M}."
2. For each correct table, find the best-matching user table by column name overlap (normalized: trimmed lowercase). If no match → fail with "Missing table with columns: {cols}."
3. For matched pair: check column names (set-equal, case-insensitive). Diff: missing = correct columns not in user, extra = user columns not in correct.
4. Check PK assignment: columns marked isPK must set-equal the correct table's PK columns (case-insensitive).
5. Check FDs: each correct FD's lhs and rhs must appear in user's FDs (set-equal, case-insensitive). Missing FDs → fail.
6. If all checks pass → `{ pass: true, explanation: 'Correct! Your table satisfies this normal form.' }`
7. Collect all failures and return the first meaningful explanation string (not a list dump — write a readable sentence).

Write a co-located test at `src/lib/normalizationValidator.test.ts` covering the behavior cases above. Use whatever test runner is already configured (check package.json). If no test runner is configured, write the tests as Jest/Vitest compatible (vitest is preferred if not present).
