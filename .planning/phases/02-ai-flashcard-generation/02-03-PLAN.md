  <action>
Replace src/app/flashcards/page.tsx with a client component that reads generatedCards from the store:

```tsx
'use client'
import { StudySession } from '@/components/StudySession'
import { FLASHCARDS } from '@/data/flashcards'
import { useFlashcardStore } from '@/stores/flashcardStore'

export default function FlashcardsPage() {
  const generatedCards = useFlashcardStore((s) => s.generatedCards)
  const cards = generatedCards.length > 0 ? generatedCards : FLASHCARDS
  return <StudySession cards={cards} />
}
```

Key reasoning:
- Must be 'use client' because useFlashcardStore is a client-side hook
- selector (s) => s.generatedCards subscribes the component  re-renders automatically when StorageHydration calls rehydrate() on initial load
- On client-side navigation from /generate (normal flow), the store is already populated when this page renders  no re-render needed
- Falls back to FLASHCARDS when generatedCards is empty (first-time users, or after clearGeneratedCards is called)
- DO NOT add useEffect + useState for hydration  the Zustand selector subscription handles this automatically
